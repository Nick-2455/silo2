# Silo Architecture

Silo is intentionally small: it reads Engram, writes Markdown, and lets humans curate that Markdown in Obsidian.

## Core principle

Engram is the source of truth.

Silo is not a memory database. It is a projection layer:

```text
Engram → Raw → Curated → Profile → Outputs
```

## Layers

### 1. Engram

Engram stores durable observations and project context.

Silo talks to Engram through `internal/engram.Client`:

```go
type Client interface {
    Search(ctx context.Context, query string) ([]Observation, error)
    Context(ctx context.Context, project string) ([]Observation, error)
}
```

Implementations:

- `MockClient`: deterministic offline data when no endpoint is configured.
- `HTTPClient`: real `engram serve` HTTP adapter.

The rest of Silo consumes `[]engram.Observation` and does not care where observations came from.

### 2. Raw

Raw notes live at:

```text
vault/Raw/Observations/
```

Generated by:

```bash
silo sync
```

Raw is a mechanical projection of Engram observations. It is safe for Silo to regenerate these files because humans should not treat Raw as the curated writing surface.

Responsibility:

- stable filenames
- safe slugs
- deterministic Markdown
- source metadata
- no interpretation

Code:

- `internal/markdown/observation.go`
- `cmd/silo/main.go` (`runSync`)

### 3. Curated

Curated notes live at:

```text
vault/Curated/
├── Architecture/
├── Projects/
├── Identity/
└── Career/
```

Generated by:

```bash
silo curate
```

Curated is the human synthesis layer. Silo only seeds missing notes. It never overwrites existing files.

Why no overwrite?

Because Curated is where the human adds meaning: summaries, corrections, prioritization, and judgment. Once a person edits a Curated note, Silo must not replace that work with generated content.

Responsibility:

- group observations by `topic_key` or type
- create seed notes with links back to Raw observations
- preserve human edits forever

Code:

- `internal/markdown/curated.go` — render seed notes
- `internal/obsidian/vault.go` — `WriteNoteIfAbsent`
- `cmd/silo/main.go` (`runCurate`)

### 4. Profile

Profile notes live at the vault root:

```text
Identity.md
Skills.md
Projects.md
Outputs.md
```

Generated by:

```bash
silo profile
```

Profile uses Curated as the primary source when useful human content exists. If Curated is missing or only contains TODO placeholders, Silo falls back to Raw/Engram observations.

Useful Curated content means:

- frontmatter ignored
- TODO placeholders ignored
- headings alone ignored
- `Related Observations` link lists ignored
- real prose or useful bullet points in `Summary` / `Notes` count

Silo maps useful Curated notes to synthetic observations:

```text
id      = curated:<relative-path>
type    = curated
title   = first H1
content = useful markdown body
project = current project
```

Code:

- `internal/curated/reader.go`
- `cmd/silo/source.go`
- `internal/identity/service.go`
- `internal/markdown/renderer.go`

### 5. Outputs

Professional outputs live at:

```text
vault/Outputs/
├── CV.md
├── LinkedIn.md
└── ProfessionalBio.md
```

Generated by:

```bash
silo outputs
```

Outputs use the same source selection as Profile:

```text
Curated if useful → otherwise Raw/Engram fallback
```

They are written with `WriteNoteIfAbsent`, so Silo seeds them once and never overwrites human edits.

Code:

- `internal/markdown/professional_outputs.go`
- `cmd/silo/source.go`
- `cmd/silo/main.go` (`runOutputs`)

## Boundaries

### `internal/engram`

Boundary to Engram.

- Converts wire formats into `engram.Observation`.
- Keeps HTTP details isolated.
- Provides mock fallback.

### `internal/markdown`

Pure rendering.

- No filesystem writes.
- No network.
- No current time.
- Deterministic output from input structs.

### `internal/obsidian`

Filesystem boundary.

- Validates paths.
- Prevents traversal.
- Writes notes.
- Enforces no-overwrite behavior through `WriteNoteIfAbsent`.

### `internal/curated`

Read-only parser for human Curated notes.

- Finds useful human content.
- Ignores generated placeholders.
- Produces synthetic observations for Profile/Outputs.

### `internal/identity`

Identity model and deterministic heuristics.

- Builds an `identity.Identity` from observations.
- Does not know whether observations came from Engram or Curated except for Evidence labels.

## Why Markdown as projection layer?

Markdown is useful here because it is:

- human-readable
- diffable
- editable in Obsidian
- portable
- deterministic to generate
- easy to delete/regenerate when desired

Markdown is not treated as the canonical memory store. Engram keeps that role.

## Idempotence model

Silo avoids generated timestamps and random ordering.

- Raw generation is deterministic.
- Curated generation is create-only.
- Profile rendering is deterministic and may rewrite root profile files.
- Outputs generation is create-only.

This keeps repeated CLI runs safe and predictable.
